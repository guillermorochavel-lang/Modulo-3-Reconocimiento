
MODULO-3-RECONOCIMIENTO

PARA LA PRACTICA 3 DEL MODULO 3 SOBRE RECONOCIMIENTO.

SE MUESTRAN LOS DOS DIFERENTES TIPOS DE REOCONOCIMIENTO EN LA CIBERSEGURIDAD.

EL RECONOCIMIENTO PASIVO, EN EL CUAL SE INVESTIGA AL INTEREACTUAR CON INFORMACION PUBLICA EN INTERNET SIN TENER UN OBJETIVO.

POR OTRO LADO, EL RECONOCIMIENTO ACTIVO, ES LA INVESTIGACION POR MEDIO DE LA INTERACCION CON UN SISTEMA OBJETIVO PARA OBTENER INFORMACION DEL SISTEMA.


PARA CONOCERLAS A PROFUNDIDAD HICIMOS USO DE 3 HERRAMIENTAS.

1. GOOGLE: DONDE A TRAVES DE DORKS PUDIMOS FILTRAR Y BUSCAR INFORMACION, SIN UN OBJETIVO CLARO.
   ALGUNDOS DE LOS DORKS QUE SE UTILIZARON FUE MI NOMBRE COMPLETO, Y DESCRUBRI QUE SI HAY MUCHA INFORMACION EN LA RED SOBRE MI, ESTO INDICA QUE LA INFORMACION PUBLICA
   PUEDE COMPROMETER LA IDENTIDAD DE MUCHAS PERSONAS A TAN SOLO UN CLICK DE DISTANCIA.

2. EXPLOIT DATABASE: IGUALMENTE FUE RECONOCIMIENTO PASIVO, AL FILTRAR MEDIANTE DORKS  NUESTRA BUSQUEDA EN UNA BASE DE DATOS PUBLICA.
   AQUI SE INTENTO OBTENER PROGRAMAS QUE NOS AYUDARAN A HACKEAR LA UBICACION DE UN TELEFONO, POR EJEMPLO. PERO ME SORPRENDIO QUE NO ENCONTRAMOS NADA
   ESTO QUIERE DECIR QUE A PESAR DE SER UNA HERRAMIENTA PUBLICA SE TIENE CIERTO CONTROL SOBRE LO QUE SE COMPARTE.

3. MALTEGO: UNA HERRAMIENTA QUE EJEMPLIFICA EL RECONOCIMIENTO ACITIVO, PUES BUSCAMOS INFORMACION DE UN SISTEMA OBJETIVO AL INTERACTUAR DIRECTAMENTE CON SUS DATOS TECNICOS.
   FSJFL

CON ESTA PRACTICA LOGRAMOS EJEMPLIFICAR LOS TIPOS DE BUSQUEDAS Y TAMBIEN CONOCER LOS RIESGOS QUE CONLLEVA EL INVESTIGAR LA INFORMACION SIN AUTORIZACION.
TAL ES EL CASO DEL RECONOCIMIENTO ACTIVO, PUES EN MALTEGO ES NECESARIO TENER EL PERMISO O TENER UNA CLAVE DE ACCESO QUE SOLO TIENEN AUTORIDADES,
PUES AL CONOCER LOS DATOS TECNICOS DE ALGUN SISTEMA SE PUEDEN CONOCER LAS VULNERABILIDADES Y A TRAVES DE ELLAS PODER LLEGAR A INFORMACION SENSIBLE DE DICHO SISTEMA QUE PODRIA
COMPROMETER LA IDENTIDAD DE UNA PERSONA, EMPRESA, ETC.
